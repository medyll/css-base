# S1-01 — Demo scaffold & shared layout

**Epic:** Demo Infrastructure
**Priority:** P0 (blocker for all other stories)
**Role:** Developer

## User Story
As a developer, I want a shared HTML template and index page so that all demos are discoverable and consistent.

## Acceptance Criteria
- [ ] `dist/demo/index.html` — hub page listing all 5 demos with links and preview thumbnails
- [ ] Shared nav snippet in each demo page linking to all other demos
- [ ] Each demo loads `../../app.css` + its own `skin.css`
- [ ] No external CDN dependencies (except optional Google Fonts)
- [ ] Folder structure: `dist/demo/{netflix,amazon,whatsapp,windows,macos,spotify,slack,github,dashboard,terminal}/index.html + skin.css`
- [ ] **Theme toggle:** shared nav includes a light/dark toggle button that sets `data-theme="light|dark"` on `<html>` (minimal inline JS: ~3 lines)
- [ ] Toggle persists via `localStorage` so reloads keep the chosen mode

## Notes
- The nav bar should use `css-base` components (`.btn`, `.badge`, utility classes)
- Index page is itself a demo of `css-base` default styling
- The theme toggle is critical — it's the primary tool for debugging the skin system

## Validation Workflow (applies to ALL demos)

After each demo is built, the developer MUST:
1. **Playwright screenshot** — light mode + dark mode (1920x1080 viewport)
2. **Visual check** — review screenshots for broken layouts, unreadable text, missing styles
3. **If css-base issue found** — fix `dist/skill/css-base/SKILL.md`, then run `/skiller` to redeploy
4. **Screenshot saved** to `bmad/artifacts/screenshots/{demo-name}-{light|dark}.png`
